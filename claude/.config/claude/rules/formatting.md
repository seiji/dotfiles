# Formatting Rules

## General
- Prefer consistency within existing files
- Use appropriate tools for auto-formatting when available

## JSON Files
- Sort object keys alphabetically (ASCII order)
- Use 2 spaces for indentation
- No trailing commas
- Applies to: `devcontainer.json`, `package.json`, `tsconfig.json`, etc.

## YAML Files
- Sort keys alphabetically where appropriate
- Use 2 spaces for indentation
- Applies to: GitHub Actions, Docker Compose, etc.

## Configuration Files
- Keep related settings grouped when sorting would reduce readability
- Add comments to explain non-obvious configurations
