# Coding Standards

## General
- Follow existing code style in the project
- Keep changes minimal and focused
- Avoid over-engineering

## Code Quality
- Prefer readability over cleverness
- Don't add unnecessary comments for self-explanatory code
- Remove unused code instead of commenting it out

## Changes
- Ask before making destructive or large-scale changes
- Explain significant changes before implementing
- Run tests after code changes when available
